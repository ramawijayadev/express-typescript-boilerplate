0. Legend

  • CORE Mandatory. Application will not run or will violate architecture standards without this.
  • RECOMMENDED Highly Suggested. "Production Ready" standard. Can be replaced only if there is a very strong reason.
  • OPTIONAL Optional. Use only if the specific feature is required.

1. Tech Baseline & Principles

We choose technologies that are stable, typed, and mainstream to facilitate onboarding.

  • Package Manager: pnpm (Mandatory, for speed and disk efficiency).
  • Runtime: Node.js (LTS Version).
  • Framework: Express (Industry standard).
  • Language: TypeScript (Strict mode).
  • Database: PostgreSQL with Prisma ORM.
  • Testing: Vitest (Faster replacement for Jest).

2. Core Runtime & HTTP

express CORE

Minimalist web framework for routing and middleware.

zod CORE

Schema validation. Mandatory for validating env vars and request body/params.

dotenv CORE

Loads environment variables from a .env file.

axios RECOMMENDED

Promise-based HTTP Client for external API requests.

uuid RECOMMENDED

Unique ID generator (UUID v4) for non-database purposes (e.g., Request ID).

http-status-codes RECOMMENDED

Standard constants for HTTP status codes (e.g., StatusCodes.OK).

cookie-parser RECOMMENDED

Middleware to parse cookie headers and populate req.cookies.

3. Security & HTTP Hardening

helmet CORE

Secures the app by setting various HTTP headers (HSTS, XSS Filter, etc.).

cors CORE

Middleware to enable Cross-Origin Resource Sharing.

express-rate-limit RECOMMENDED

Prevents brute-force/simple DDoS attacks by limiting requests per IP.

hpp RECOMMENDED

Protection against HTTP Parameter Pollution attacks.

4. Auth & Crypto

jsonwebtoken CORE

Industry standard for stateless authentication (Access Token & Refresh Token).

argon2 CORE

Competition-winning password hashing algorithm, more secure and GPU-resistant than bcrypt.

5. Database & Migrations

@prisma/client CORE

Auto-generated & type-safe query builder for Node.js.

prisma CORE (Dev)

CLI tool for schema declaration, migration management, and client generation.

pg CORE

PostgreSQL client for Node.js (native driver).

@prisma/adapter-pg CORE

Driver adapter to use the `pg` driver with Prisma (serverless ready).

6. Cache & Queue

ioredis CORE

Robust Redis client supporting clusters, used for caching and sessions.

bullmq RECOMMENDED

Redis-based Message Queue. Mandatory for heavy/async processes (e.g. sending emails, generating PDFs).

7. Logging & Observability

pino CORE

Low-overhead logger with JSON output (Mandatory for production tracing).

pino-roll CORE

Log rotation utility for pino, ensuring log files don't grow uncontrollably.

pino-pretty RECOMMENDED (Dev)

Formats JSON logs into readable colored text during development.

8. Mailing System

nodemailer RECOMMENDED

Standard module for sending emails via SMTP.

9. Real-time Communication

socket.io OPTIONAL

WebSocket engine with fallback and room management features. Use if live chat/notifications are needed.

10. API Documentation

swagger-ui-express RECOMMENDED

Serves the visual Swagger/OpenAPI interface at the /docs endpoint.

swagger-jsdoc RECOMMENDED

Generates OpenAPI specifications automatically from JSDoc comments in code (Legacy support).

@asteasolutions/zod-to-openapi CORE

Generates OpenAPI definitions directly from Zod schemas. Guarantees documentation is always in sync with validation.

11. Testing

vitest CORE (Dev)

Next-generation test runner (Vite-native). Faster than Jest, minimal configuration.

supertest RECOMMENDED (Dev)

HTTP assertion library for Integration and E2E Tests.

12. TypeScript & Build Tooling

All dependencies below are devDependencies:

  • typescript: Main compiler.
  • tsx: Executes TypeScript files directly (faster replacement for ts-node).
  • rimraf: Folder deletion utility (rm -rf) that is cross-platform.
  • concurrently: Runs multiple commands in parallel (e.g. server + worker).
  • cross-env: Sets environment variables across platforms (Windows/Linux).
  • nodemon: Utility to monitor file changes and restart server automatically.
  • Type definitions: @types/node, @types/express, etc.

13. Linting, Formatting & DX

eslint CORE

Static analysis to find code issues (bugs, unused vars).

prettier CORE

Automatic code formatter. Eliminates tabs vs spaces debates.

husky & lint-staged RECOMMENDED

Git hooks to run lint/test automatically before commit.

14. Initial Installation

Copy the commands below to install the entire standard stack in one go:

Production Dependencies

pnpm add express zod dotenv axios uuid \
helmet cors express-rate-limit hpp cookie-parser \
jsonwebtoken argon2 \
@prisma/client ioredis bullmq \
pino pino-pretty \
nodemailer \
socket.io \
swagger-ui-express swagger-jsdoc

Development Dependencies

pnpm add -D typescript tsx rimraf \
prisma vitest supertest \
eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin \
eslint-config-prettier prettier \
husky lint-staged cross-env concurrently \
@types/node @types/express @types/cors @types/cookie-parser \
@types/swagger-jsdoc @types/swagger-ui-express @types/uuid

15. External Tools (Non-NPM)

The following tools are not installed via package.json, but must be present in the environment (Local/CI):

Docker & Docker Compose

Runs infrastructure dependencies (PostgreSQL, Redis) in isolation without polluting the host OS.

k6

Load testing tool (written in Go/JS). Used for API stress tests (see tests/perf folder).
